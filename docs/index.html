alc-hub CoreS3

M5Stack CoreS3 用 点呼キオスク統合ハブ ファームウェア書き込みツール

対応機器

マイコン M5Stack CoreS3
アルコールチェッカー FC-1200 (RS232M Module 経由)
体温計 ニプロ NT-100B (BLE)
血圧計 ニプロ NBP-1BLE (BLE)

書き込み手順

  1. CoreS3 を USB-C ケーブルで PC に接続
  2. 下の「Install」ボタンをクリック
  3. シリアルポートを選択（COM○ または /dev/ttyACM○）
  4. 書き込み完了を待つ（1〜2分）

動作要件

  • Chrome, Edge, Opera のいずれか（Web Serial API 対応ブラウザ）
  • CoreS3 は USB Serial/JTAG 内蔵のため、通常は追加ドライバ不要（Windows 10 以降 / macOS / Linux）

注意事項

  • 書き込み中は USB ケーブルを抜かないでください
  • ポートが認識されない・書き込みが始まらない場合は、リセットボタンを約2秒長押しして書き込みモード（ダウンロードモード）に入れてから再試行してください
  • 書き込み後、CoreS3 は自動的に再起動し、起動画面 → 待機画面（カードをかざしてください）が表示されます

ファームウェアをインストール

お使いのブラウザは Web Serial API に対応していません。
Chrome, Edge, または Opera をお使いください。 シリアルポートへのアクセスが許可されていません。

Wi-Fi 設定

本ファームウェアは Improv Wi-Fi Serial に対応しています。 設定方法は 2 つ（どちらも結果は同じ。NVS に保存され、以後は起動時に自動接続）:

  • 下の「Wi-Fi 設定（いつでも）」フォーム — この説明を読みながら設定できます
  • インストールダイアログの「Wi-Fi 設定 (Configure Wi-Fi)」 — 書き込み完了後に表示（周辺ネットワークの選択も可能）

※ ダイアログに Wi-Fi 項目が出ない場合は、ファームウェアが Improv 対応前の版です。先に「Install」で最新版に更新してください。

設定できるネットワークの条件

  • 周波数: 2.4GHz 帯のみ（ESP32-S3 は 5GHz / 6GHz 非対応。IEEE 802.11 b/g/n）
  • パスワード: WPA / WPA2 / WPA3-Personal（8〜63 文字の半角英数記号）、またはパスワード無しのオープンネットワーク
  • SSID は最大 32 バイト（日本語 SSID は非推奨）
  • WPA2/WPA3-Enterprise（802.1X 認証）や Web ログイン式（キャプティブポータル）は非対応

※ Wi-Fi は LAN 配線が無い拠点向けの代替経路です（主経路は LAN Module 13.2 / PoE）。 設定状態は本体の「ログ確認」画面（WiFi インジケータ・IP アドレス）で確認できます。

Wi-Fi 設定（いつでも）

SSID とパスワードを入力して「接続して設定」を押し、CoreS3 のシリアルポートを選択してください。 接続確認まで最大 30 秒ほどかかります。

※ 上の「設定できるネットワークの条件」を満たすネットワークのみ接続できます。 インストールダイアログや他のシリアルモニタが同じポートを開いていると失敗します（先に閉じてください）。

画面向き設定（インストール後）

設置向き（壁掛け・逆さ付けなど）に合わせて画面の回転を設定できます。 設定は本体（NVS）に保存され、再起動後も維持されます。

※ ファームウェア書き込みと同じ USB 接続のまま実行できます（Install ダイアログを閉じてから）。 シリアルモニタからは ROTATE 180 のようにコマンド送信しても同じ設定ができます。

書き込み後の動作確認

  • 画面: 起動画面の後、待機画面「カードをかざしてください」とステータスバー (LAN / 232 / BLE / WiFi + 稼働時間) が表示されます
  • タッチ: 待機画面タップ → メニュー（上: 点呼 / 下: ログ確認）。各画面はタップで待機画面に戻ります
  • 体温・血圧: ニプロ NT-100B / NBP-1BLE で測定すると、測定値が画面に大きく表示され、シリアルにも {"type":"temperature",...} 形式の JSON が出力されます (ble-medical-gateway 互換)
  • シリアル (115200bps): PING 送信 → PONG 応答、QR https://example.com 30 で QR 表示をテストできます
